# 生计项目 - HTTP 局域网测试安全注意事项

## 重要安全提醒

在局域网环境下，由于浏览器限制，无法使用 Web Crypto API（仅在 HTTPS 或 localhost 环境下可用）。因此，我们改用 crypto-js 库进行前端密码哈希处理。

现在在 HTTP 环境下也可以进行密码哈希处理，提升了安全性。

## 当前安全措施

1. **前端哈希处理**：使用 crypto-js 库在前端对密码进行 SHA256 哈希
2. **后端二次哈希**：后端接收哈希值后使用 bcrypt 进行二次哈希存储
3. **双重保护**：前端 SHA256 + 后端 bcrypt 双重哈希机制

## 风险说明

虽然现在有前端哈希处理，但在 HTTP 环境下仍存在一定风险：
1. **中间人攻击风险**：恶意用户可能截获网络流量获取哈希值
2. **重放攻击风险**：攻击者可能使用截获的哈希值进行重放攻击

## 安全建议

### 生产环境部署
1. 必须使用 HTTPS 协议
2. 可继续使用当前的前端 SHA256 + 后端 bcrypt 双重哈希机制
3. 实施 HSTS（HTTP Strict Transport Security）
4. 添加适当的安全头（CSP、X-Frame-Options 等）

### 局域网测试期间
1. 在受信任的网络环境中进行测试
2. 使用非敏感密码进行测试
3. 限制访问权限，避免外部访问

## 代码变更记录

### 前端变更
- 在 Login.vue 和 Register.vue 中引入 crypto-js 库
- 使用 SHA256 对密码进行前端哈希处理
- 保留后端 bcrypt 二次哈希处理

### 依赖安装
- 安装 `crypto-js` 库用于前端加密

## 生产环境安全机制

在生产环境中，密码会经历以下处理流程：
1. 用户输入明文密码
2. 前端使用 SHA256 对密码进行哈希
3. 传输到后端（通过 HTTPS 加密传输）
4. 后端使用 bcrypt 对哈希值进行二次哈希并存储

这种双重哈希机制确保即使某个环节出现问题，密码仍然难以被破解。

## 总结

通过引入 crypto-js 库，我们能够在 HTTP 环境下实现前端密码哈希处理，显著提升了安全性。这种方案在局域网测试和生产环境中都适用，只是生产环境还需要 HTTPS 传输加密作为额外保护层。

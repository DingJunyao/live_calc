# 详情页宽屏响应式布局

## 概述

为菜谱详情、原料详情、商品详情三个页面实现了宽屏（≥960px）响应式双栏布局，移动端保持原有顺序。

## 布局方案

采用 **CSS Grid + Flexbox 混合布局**：
- **行对齐区域**（同一行需要左右对齐）：使用 CSS Grid 的 `grid-row`/`grid-column` 显式定位
- **行不对齐区域**（左右列高度独立）：使用 Flex 容器包裹，`align-items: flex-start` 使列高度互不影响

### 移动端处理

- 使用 `display: contents` 使 Flex 容器的子元素直接参与父级 flex 流
- 通过 CSS `order` 属性控制移动端卡片顺序
- 容器内卡片的 margin 通过 `.group-xx-left > .grid-item` 选择器匹配（因为 `display: contents` 不影响 CSS 选择器）

## 各页面布局

### 菜谱详情 (RecipeDetail.vue)

```
桌面端：
基本信息      | 最新价格          ← 对齐
原料列表      | 价格趋势          ← 对齐
──────────────────────────────────
相关商品      | 步骤              ← 对齐
──────────────────────────────────
营养成分      | 备注              ← 不对齐（独立列）
              | 标签              ← 不对齐（独立列）
```

### 原料详情 (IngredientDetail.vue)

```
桌面端：
基本信息      | 最新价格          ← 对齐
关联商品      | 价格趋势          ← 对齐
──────────────────────────────────
相关菜谱      | 价格记录          ← 对齐
──────────────────────────────────
营养成分      | 层级关系          ← 不对齐（独立列）
              | 关系列表          ← 不对齐（独立列）
              | 单位与密度        ← 不对齐（独立列）
```

移动端顺序：基本信息 → 最新价格 → 价格趋势 → 关联商品 → 价格记录 → 相关菜谱 → 营养成分 → 层级关系 → 单位与密度

### 商品详情 (ProductDetail.vue)

```
桌面端：
基本信息      | 最新价格          ← 对齐
单位与密度    | 价格趋势          ← 不对齐（独立列）
──────────────────────────────────
营养成分      | 价格记录          ← 对齐
```

移动端顺序（不变）：基本信息 → 最新价格 → 价格趋势 → 价格记录 → 营养成分 → 单位与密度

## 其他改动

- 原料详情：关联菜谱添加了分页器，与价格记录列表样式一致（`lines="two"`）
- 对话框（单位、密度、编辑等）移到布局容器外部，避免影响 Grid 布局

## 关键技术点

1. **`display: contents`**：在移动端让 wrapper div "透明"，子元素直接参与父级 flex 流，同时允许 CSS `order` 重排序
2. **`display: contents` 下的 margin 问题**：`.parent > .grid-item` 选择器无法匹配 `display: contents` 容器内的子元素，需要额外添加 `.group-xx-left > .grid-item` 等选择器
3. **营养成分展开不影响右列**：不对齐区域使用 Flex 容器而非 CSS Grid row，避免行轨道高度变化影响右列

# 多商品成本计算修复

## 问题描述

菜谱 347（火腿肠炒黄瓜）中的「火腿肠」在菜谱详情页显示成本为 `-`（无成本），但在食材详情页能正常显示价格。

## 根本原因

食材（ingredient_id=240，火腿肠）关联了多个商品：

| 商品 | 价格记录数 |
|------|:---:|
| 火腿肠（通用） | 0条 |
| 双汇王中王 | 1条 |
| 双汇弹脆肠 | 1条 |
| 金锣香甜王 | 1条 |
| 金锣好口福 | 1条 |
| 金锣肉粒多肉花肠 | 1条 |
| 金锣王特级火腿肠 | 1条 |

菜谱成本计算函数（`calculate_recipe_cost`、`calculate_recipe_cost_range_as_of`、`calculate_recipe_cost_as_of`）在 `recipe_service.py` 中只取了 `Product.ingredient_id == ingredient.id` 的**第一个商品**（通用名称"火腿肠"，0条记录），那些有价格记录的品牌商品被忽略了。

而食材详情页的「最新价格」API 查询了**所有商品**的价格记录，所以能展示价格。

## 修复方案

修改 `recipe_service.py` 中的三个成本计算函数，当查找价格记录时遍历该食材下的**所有商品**，而非仅取第一个。

### 具体变更

1. **`_get_price_records_for_date`** — 新增 `product_id` 参数，允许调用方指定要查询的商品 ID
2. **`calculate_recipe_cost`** — 遍历所有商品查找当天价格记录，遍历所有商品做前向填充
3. **`calculate_recipe_cost_range_as_of`** — 同上
4. **`calculate_recipe_cost_as_of`** — 同上

**Why:** 食材和商品是一对多关系（一个食材可能有多个品牌商品），只取第一个商品可能拿到无价格记录的商品。遍历所有商品可以找到有价格记录的那个。

**How to apply:** 该修复自动生效。如果以后遇到类似问题（食材详情有价格但菜谱成本显示 -），优先检查该食材是否关联了多个商品，以及第一个商品是否有价格记录。

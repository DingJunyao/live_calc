# 开发时的一些规则

## 前端

使用 Vuetify 框架，需要兼顾桌面端和移动端的用户体验。控件也优先选用其提供的控件。

所有页面都有导航栏，其左上方一定是汉堡按钮，负责显示、隐藏侧边栏。第二个按钮如果有的话，一般是后退按钮。

由于一些浏览器会自动忽略重复出现的 alert、confirm，故整个项目都不能使用浏览器层面的提示，而是改用模态框。

项目里面所有场合都不要使用 emoji。需要的地方请使用 Vuetify 提供的图标代替。

## 后端

所有的删除操作都是软删除。

凡是遇到错误，应该输出以下信息，便于排查：

- 请求详情，包括请求 URL、请求方式、请求体、响应体等信息
- 异常堆栈

而传到前端的错误信息，必须是一般人能够读懂的，而非抛出错误堆栈、HTTP 错误码。

后端目前有很多依赖管理方式。如果修改了依赖项，需保证全部的这些方式都要一并更改。

## 数据库

所有的数据表都需要有以下几条：

- 创建时间
- 修改时间
- 创建者
- 修改者
- 是否有效

后端必须同时兼容 SQLite、MySQL、PostgreSQL。

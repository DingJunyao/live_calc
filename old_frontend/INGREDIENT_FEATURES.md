# 食材层级关系与合并功能说明

## 概述

本项目已实现食材层级关系管理和合并功能，解决了以下问题：
- 食材不当分立（如"米"和"大米"被认为是不同食材）
- 处理层级关系（如"鸡肉"、"鸡胸肉"、"鸡翅"等）
- 实现食材合并功能，将多个相似食材合并为一个
- 支持回退机制，在计算成本和营养值时找不到精确食材时向上查找
- 数据操作层面的合并，将相关记录（菜谱、价格记录等）自动重定向到合并后的食材

## 功能使用

### 1. 食材合并功能
在**原料管理**页面：
1. 选择需要合并的食材
2. 点击"合并"按钮
3. 选择目标食材
4. 确认合并操作

合并后，源食材的数据将迁移到目标食材，原食材会被标记为已合并状态。

### 2. 层级关系查看
在**原料管理**页面：
1. 选择一个食材
2. 点击"层级关系"按钮
3. 查看该食材的父子层级关系

### 3. 合并状态检查
在**原料管理**页面：
1. 选择一个食材
2. 点击"替代品"按钮
3. 查看食材的合并状态信息

## API 端点

### 合并功能
- `POST /api/v1/ingredients/merge` - 合并食材
- `GET /api/v1/ingredients/merge-history` - 获取合并历史
- `GET /api/v1/ingredients/{id}/merge-status` - 获取食材合并状态

### 层级关系功能
- `GET /api/v1/ingredients/{id}/hierarchy` - 获取食材层级关系
- `POST /api/v1/ingredients/hierarchy` - 创建层级关系
- `DELETE /api/v1/ingredients/hierarchy/{relation_id}` - 删除层级关系

## 权限控制
- 合并功能需要管理员权限
- 层级关系管理需要管理员或高级用户权限
- 普通用户只能查看层级关系和合并状态

## 注意事项
1. 合并操作是不可逆的，请谨慎操作
2. 合并后原食材会被标记为已合并状态，但仍可在历史记录中查询
3. 所有相关数据（菜谱、价格记录、营养数据等）会自动重定向到目标食材
4. 合并前请确认目标食材是合适的聚合目标

## 后端支持
此功能需要配合后端的食材层级关系与合并功能使用。如需更多信息，请参阅后端文档。

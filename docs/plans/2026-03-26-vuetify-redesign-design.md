# Vuetify 前端重写设计方案

## 项目概述

**目标**：使用 Vuetify 3 完全重写前端，打造符合用户使用习惯、移动端优先、桌面端信息密集的全新交互体验。

**核心原则**：
1. **移动端优先** - 主要使用场景是移动端，尤其价格记录功能
2. **交互驱动** - 重新设计交互流程，而非简单组件替换
3. **组件库赋能** - 使用 Vuetify 提高开发效率，降低维护成本
4. **响应式信息架构** - 不同设备展示不同信息密度，非简单放大

**设计背景**：
- 用户反馈：之前三次尝试都是"换皮"，没有真正重新设计交互
- 痛点：调整交互耗时太长，希望用组件库节省开发成本
- 核心场景：移动端维护价格（P0）、浏览菜谱（P1）

## 用户需求分析

### 使用场景

**移动端为主**（使用频率 > 80%）：
- 购物时快速记录价格
- 随时查看菜谱成本和营养
- 单手操作友好

**桌面端为辅**：
- 家中批量数据管理
- 信息密集展示
- 多任务处理

### 功能优先级

| 优先级 | 功能 | 使用频率 | 设计重点 |
|--------|------|----------|----------|
| P0 | 价格记录 | 最高 | 快速录入、单手操作 |
| P1 | 菜谱浏览 | 其次 | 查看成本、营养信息 |
| P2 | 商品/原料/商家管理 | 较少 | 列表浏览、详情查看 |
| P3 | 仪表盘/统计 | 最低 | 概览信息 |

### 交互设计诉求

**价格记录（P0）**：
- 快速录入优先（移动端单手操作）
- 大按钮、大输入框、流畅流程

**菜谱管理（P1）**：
- 浏览查看为主
- 移动端卡片式浏览
- 桌面端信息密集展示

**桌面端要求**：
- 充分利用大屏空间
- 不是移动端UI的简单放大
- 表格、多列布局、键盘快捷键

## 整体信息架构

### 移动端架构（快速操作导向）

**底部导航栏**（4个主要入口）：
```
[记录] [菜谱] [数据] [我的]
```

| 导航项 | 功能描述 |
|--------|----------|
| **记录**（首页） | 价格记录核心功能，大FAB快速入口 |
| **菜谱** | 菜谱网格卡片浏览，详情查看 |
| **数据** | 商品、原料、商家列表管理 |
| **我的** | 设置、统计、个人中心 |

### 桌面端架构（信息密集导向）

**三栏布局**：
```
[导航侧边栏] [主内容区] [详情/辅助面板]
```

| 区域 | 功能描述 |
|------|----------|
| **左侧导航栏**（固定） | 全部功能列表，可收起为图标模式 |
| **主内容区** | 数据表格/卡片网格，支持Tab切换 |
| **右侧辅助面板**（可选） | 选中项详情、快速操作、图表展示 |

### 响应式断点策略

| 断点 | 宽度 | 导航方式 | 布局 | 信息密度 |
|------|------|----------|------|----------|
| xs | < 600px | 底部导航栏 | 单列 | 精简，核心信息 |
| sm | 600-960px | 底部导航栏 | 单/双列 | 中等 |
| md | 960-1280px | 侧边导航（可收起） | 双列 | 较高 |
| lg/xl | > 1280px | 固定侧边栏 | 多列 | 高密度，辅助面板 |

**快速操作入口**：
1. **主操作按钮** - 工具栏右上角"添加记录"按钮
2. **操作按钮组** - 搜索栏下方：刷新、导出、批量操作
3. **键盘快捷键**：
   - `N` - 新建记录
   - `E` - 编辑选中行
   - `D` - 删除选中行
   - `Ctrl+F` - 聚焦搜索框
4. **表格右键菜单** - 查看详情、编辑、删除

**Vuetify 组件使用**：
- `<v-navigation-drawer>` - 左侧导航栏
- `<v-data-table>` - 数据表格
- `<v-toolbar>` - 工具栏和操作按钮组
- `<v-btn>` - 各种操作按钮（替代FAB）
- `<v-menu>` - 右键上下文菜单
- `<v-tooltip>` - 快捷键提示

## Vuetify 主题系统

### 设计理念

**温暖生活主题**：
- 🍃 **自然健康** - 绿色系为主，呼应营养、健康
- 💰 **理性消费** - 蓝色点缀，代表数据、理性
- 🏠 **居家温暖** - 米色背景，营造温馨感
- 🌙 **夜间护眼** - 深色模式降低饱和度

### 主题切换逻辑

```typescript
// 优先级：手动设置 > 系统偏好 > 默认浅色
const themeMode = useStorage('theme-mode', 'system')

// 监听系统变化
const prefersDark = usePreferredDark()

// 计算实际主题
const actualTheme = computed(() => {
  if (themeMode.value === 'system') {
    return prefersDark.value ? 'dark' : 'light'
  }
  return themeMode.value
})
```

**切换入口位置**：
- 移动端："我的"Tab → 设置 → 主题切换
- 桌面端：右上角用户菜单 → 主题切换

## 风险与缓解措施

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| ECharts 暗色模式 | 图表显示异常 | 封装组件，监听主题切换自动刷新 |
| Leaflet 样式冲突 | 地图控件异常 | CSS 命名空间隔离 |
| 表单验证逻辑 | 迁移复杂 | 使用 Vuetify 内置验证规则 |
| 移动端键盘 | 布局错乱 | 使用 Vuetify v-layout 处理 |
| 大数据量列表 | 性能问题 | 使用虚拟滚动（v-virtual-scroll） |
| 桌面端交互 | 不同平台习惯 | 提供键盘快捷键，支持多种操作方式 |
| 主题色彩不够突出 | 视觉效果不佳 | 已设计独特的温暖生活主题 |

## 决策记录

| 项目 | 决策 | 理由 |
|------|------|------|
| 组件库 | Vuetify 3 | 成熟稳定、组件丰富、文档完善 |
| 设计规范 | 自定义主题 + Material Design 3 | 避免通用AI美学，打造独特温暖风格 |
| 迁移方式 | 完全重写 | 用户反馈之前尝试都是换皮 |
| 响应式策略 | 双设备信息架构 | 移动端快速操作，桌面端信息密集 |
| 导航方式 | 底部导航(移动) + 侧边栏(桌面) | 符合各自平台使用习惯 |
| FAB使用 | 仅移动端 | 桌面端有更好的操作入口 |
| 表单模式 | 全屏(移动) + 对话框(桌面) | 不同设备最佳体验 |
| 保留组件 | ECharts、Leaflet | 功能复杂，重写成本高 |
| 预计工期 | 10 天 | 按6个阶段规划 |

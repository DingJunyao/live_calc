# 价格记录页点击跳转详情 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 价格记录页点击记录整体跳商品明细、点击记录内商家跳商家详情，三层点击互不打架。

**Architecture:** 单文件 `PricesView.vue` 纯前端改动。新增 `goToProduct` / `goToMerchant` 两个跳转方法；移动端 `v-list-item` 与桌面端 `v-card` 各加整行/整卡 `@click` 跳商品；行内商家元素 `@click.stop` 隔离冒泡并跳商家详情；原有 6 处操作按钮补 `.stop` 防误触。复用既有 `product-detail` / `merchant-detail` 路由，零后端改动。

**Tech Stack:** Vue 3 + Vuetify 3 + Vue Router（均既有）

**验证策略说明:** 前端项目无单测框架（仅 `npm run build` 校验），本改动为模板交互绑定、无可单测的纯逻辑（两方法各两行且含路由副作用），引入测试框架不划算（YAGNI）。故采用「实现 → dev 热更新即时验证 → `npm run build` 正式校验 → 手测清单」。git 提交按项目规矩不在计划内自动执行，由开发者决定。

**行号说明:** 本计划引用的行号基于编辑前的 [PricesView.vue](frontend/src/views/prices/PricesView.vue)；Task 1 插入方法后，后续模板行号会整体下移约 9 行，故模板定位一律以「before 代码块」为准、行号仅辅助。

## Task 1: 新增两个跳转方法

**Files:**
- Modify: `frontend/src/views/prices/PricesView.vue`（`<script setup>`，`const router = useRouter()` 之后，约 [:322](frontend/src/views/prices/PricesView.vue#L322)）

- [ ] **Step 1: 在 `router` 声明之后插入两个方法**

在 `const router = useRouter()`（约 [:322](frontend/src/views/prices/PricesView.vue#L322)）与 `const { isDesktop, toggleSidebar } = useMobileDrawerControl()`（约 [:323](frontend/src/views/prices/PricesView.vue#L323)）之间插入：

```ts
// 跳转：点记录整体 → 商品明细
const goToProduct = (record: PriceRecord) => {
  if (!record.product_id) return
  router.push(`/data/products/${record.product_id}`)
}

// 跳转：点记录内商家 → 商家详情
const goToMerchant = (record: PriceRecord) => {
  if (!record.merchant_id) return
  router.push(`/data/merchants/${record.merchant_id}`)
}
```

`PriceRecord` 接口已在 [:326](frontend/src/views/prices/PricesView.vue#L326) 定义（含 `product_id: number`、`merchant_id?: number`），`router` 已声明，**无需新增 import**。

- [ ] **Step 2: dev 热更新即时验证**

vite 自动重载会刷新页面，控制台应**无报错**（方法尚未被模板调用，功能此时不可见属正常）。留待 Task 4 正式 build。

## Task 2: 移动端列表项（整行 + 商家 + 按钮）

**Files:**
- Modify: `frontend/src/views/prices/PricesView.vue` 模板「移动端：列表样式」段（约 [:43-74](frontend/src/views/prices/PricesView.vue#L43-L74)）

- [ ] **Step 1: `v-list-item` 加整行 `@click`**

before:
```vue
<v-list-item v-for="record in records" :key="record.id">
```
after:
```vue
<v-list-item v-for="record in records" :key="record.id"
             @click="goToProduct(record)">
```

- [ ] **Step 2: 商家行改为可点 `span`**

定位「未知商家」所在 subtitle（原 `{{ record.merchant_name || '未知商家' }}`）。

before:
```vue
<v-list-item-subtitle>
  {{ record.merchant_name || '未知商家' }}
</v-list-item-subtitle>
```
after:
```vue
<v-list-item-subtitle>
  <span v-if="record.merchant_id"
        class="text-primary cursor-pointer"
        @click.stop="goToMerchant(record)">
    {{ record.merchant_name }}
  </span>
  <span v-else class="text-medium-emphasis">未知商家</span>
</v-list-item-subtitle>
```

- [ ] **Step 3: append 区三按钮补 `.stop`**

before:
```vue
<v-btn icon="mdi-tag-multiple" size="small" variant="text" class="mr-1" @click="openRecordAgainDialog(record)" />
<v-btn icon="mdi-pencil" size="small" variant="text" color="primary" class="mr-1" @click="openEditDialog(record)" />
<v-btn icon="mdi-delete" size="small" variant="text" color="error" @click="deleteRecord(record.id)" />
```
after:
```vue
<v-btn icon="mdi-tag-multiple" size="small" variant="text" class="mr-1" @click.stop="openRecordAgainDialog(record)" />
<v-btn icon="mdi-pencil" size="small" variant="text" color="primary" class="mr-1" @click.stop="openEditDialog(record)" />
<v-btn icon="mdi-delete" size="small" variant="text" color="error" @click.stop="deleteRecord(record.id)" />
```

- [ ] **Step 4: dev 热更新即时验证**

浏览器开窄屏（<600px）看 `/prices`：点列表项空白处应跳商品详情（热更新后即可见），点商家应跳商家详情，点三按钮只做各自操作。控制台无报错。

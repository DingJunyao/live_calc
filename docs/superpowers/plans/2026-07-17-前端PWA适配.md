# 前端 PWA 适配 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让「生计」前端成为可安装的 PWA（最小集）——可装到桌面/主屏、app shell 预缓存二次打开秒开、断网可打开壳子、prompt 模式可控更新。

**Architecture:** 用 `vite-plugin-pwa`（底层 Workbox）自动生成 `manifest.webmanifest` 与 service worker；思源黑体转曲生成路径化 `logo.svg`（字体不入仓库），`@vite-pwa/assets-generator` 从 SVG 产出全套图标；新增 `PWAUpdatePrompt.vue` 组件做更新提示；nginx 模板补 SW/manifest 的 no-cache header。

**Tech Stack:** Vue 3 + Vite 5 + TypeScript + Vuetify 3；`vite-plugin-pwa`；`@vite-pwa/assets-generator`；Python `fontTools`（仅生成 logo 用，复用根目录 `.venv`）；nginx。

## Task 2: 生成路径化 logo 源 SVG（思源黑体转曲）

**Files:**
- Create: `scripts/generate_pwa_logo.py`
- Create: `frontend/src/assets/logo.svg`（脚本产出）
- Modify: `.gitignore`

**说明**：思源黑体仅在生成 logo 时用一次，把「生计」二字字形 dump 为 SVG `<path>`（转曲）。字体文件不入仓库，脚本入仓库便于将来换 logo 重跑。

- [ ] **Step 1: 创建生成脚本**

新建 `scripts/generate_pwa_logo.py`：

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-
"""
生成 PWA logo SVG（路径化 / 转曲）。

用思源黑体 SC Bold 把「生计」二字字形提取为 SVG <path>，
横排居中于 1024×1024 橄榄绿背景画布。转曲后 SVG 不依赖字体，
字体文件仅本脚本运行时使用，不入仓库（scripts/.fonts/ 已 gitignore）。

使用：
  1. 下载思源黑体 SC Bold OTF：
     https://github.com/adobe-fonts/source-han-sans/releases
     选 SourceHanSansSC.zip，解压取 SourceHanSansSC-Bold.otf
  2. 放到 scripts/.fonts/SourceHanSansSC-Bold.otf
  3. 在项目根用 .venv 运行：python scripts/generate_pwa_logo.py
  4. 产出 frontend/src/assets/logo.svg
"""
from pathlib import Path
from fontTools.ttLib import TTFont
from fontTools.pens.svgPathPen import SvgPathPen

# ---- 配置 ----
FONT_PATH = Path(__file__).parent / ".fonts" / "SourceHanSansSC-Bold.otf"
TEXT = "生计"
OUT_SVG = Path(__file__).parent.parent / "frontend" / "src" / "assets" / "logo.svg"

VIEWBOX = 1024            # 画布尺寸
BG_COLOR = "#558B2F"      # 橄榄绿 primary（对齐 vuetify.ts lightTheme）
FG_COLOR = "#FDFCF8"      # 温暖米白
FONT_SIZE = 560           # 字号；两字横排约占画布宽 55%，四周留白兼容 maskable 安全区


def main() -> None:
    if not FONT_PATH.exists():
        raise SystemExit(
            f"字体不存在：{FONT_PATH}\n"
            "请从 https://github.com/adobe-fonts/source-han-sans/releases "
            "下载思源黑体 SC Bold OTF 放到该路径（目录已 gitignore）。"
        )

    font = TTFont(FONT_PATH)
    glyph_set = font.getGlyphSet()
    cmap = font.getBestCmap()
    units_per_em = font["head"].unitsPerEm
    scale = FONT_SIZE / units_per_em

    # 取每个字的 glyph 名 + 水平 advance，计算总宽与起始 x（水平居中）
    glyphs = []
    for ch in TEXT:
        if ord(ch) not in cmap:
            raise SystemExit(f"字体缺少字符：{ch}")
        gname = cmap[ord(ch)]
        adv = font["hmtx"][gname][0]
        glyphs.append((gname, adv))

    total_width = sum(adv for _, adv in glyphs) * scale
    x_cursor = (VIEWBOX - total_width) / 2
    baseline = VIEWBOX * 0.78  # 字底基线 y（目视居中略偏下）

    paths = []
    for gname, adv in glyphs:
        pen = SvgPathPen(glyph_set)
        glyph_set[gname].draw(pen)
        d = pen.getCommands()
        # 字体坐标 y 向上 → SVG y 向下：translate 到字位 + scale(s,-s) 翻转
        paths.append(
            f'    <path transform="translate({x_cursor:.2f},{baseline:.2f}) '
            f'scale({scale:.6f},-{scale:.6f})" d="{d}"/>'
        )
        x_cursor += adv * scale

    svg = (
        '<?xml version="1.0" encoding="UTF-8"?>\n'
        f'<svg xmlns="http://www.w3.org/2000/svg" '
        f'viewBox="0 0 {VIEWBOX} {VIEWBOX}" width="{VIEWBOX}" height="{VIEWBOX}">\n'
        f'  <rect width="{VIEWBOX}" height="{VIEWBOX}" fill="{BG_COLOR}"/>\n'
        f'  <g fill="{FG_COLOR}">\n'
        f'{chr(10).join(paths)}\n'
        f'  </g>\n'
        f'</svg>\n'
    )

    OUT_SVG.parent.mkdir(parents=True, exist_ok=True)
    OUT_SVG.write_text(svg, encoding="utf-8")
    print(f"已生成 {OUT_SVG}（路径化，{len(TEXT)} 字，无字体引用）")


if __name__ == "__main__":
    main()
```

}

}

## Task 7: nginx 模板补 SW / manifest no-cache

**Files:**
- Modify: `deploy/nginx/default.conf.template`

- [ ] **Step 1: 在 server 块靠前位置加两个精确匹配 location**

编辑 `deploy/nginx/default.conf.template`，在 `location /api/` 块**之前**（`server` 块开头，`root`/`index`/`gzip`/`client_max_body_size` 之后）插入：

```nginx
    # service worker 与 manifest 绝不能被长缓存，否则 SW 更新永远不生效
    # （= 精确匹配优先级高于下方 *.js 的正则长缓存规则）
    location = /sw.js {
        add_header Cache-Control "no-cache, no-store, must-revalidate";
        expires off;
    }
    location = /manifest.webmanifest {
        add_header Cache-Control "no-cache";
        expires off;
    }
```

> 必须排在 `location ~* \.(?:js|css|...)$`（第 35 行附近）之前——`=` 精确匹配优先级高于 `~*` 正则，但放在前面更清晰、可读性更好。

- [ ] **Step 2: 语法验证（envsubst 后 nginx -t）**

nginx 模板含 `${BACKEND_URL}` 占位，需先 envsubst 再测。在项目根执行：

```bash

## Task 8: 部署文档（deploy.md 加 PWA / HTTPS 节）

**Files:**

- [ ] **Step 1: 在 deploy.md 追加「PWA / HTTPS 部署」节**

编辑 `docs/admin/deploy.md`，在文末（或合适的部署章节后）追加：

```markdown
## PWA / HTTPS 部署

本应用已适配 PWA（可安装到桌面/主屏）。PWA 生效需注意：

### 1. 必须 HTTPS

PWA 的 service worker 仅在**安全上下文**下注册。开发环境 `localhost` 视为安全上下文，可直接测试安装；**生产环境必须 HTTPS**。

- Caddy 自动 TLS（推荐，最省事）：在 Caddyfile 配置域名，Caddy 自动申请 Let's Encrypt 证书
- nginx + Let's Encrypt：用 certbot 申请证书，nginx 监听 443 并启用 ssl

### 2. service worker / manifest 不可长缓存

`sw.js` 与 `manifest.webmanifest` 必须设为不缓存，否则浏览器死缓存旧 SW，新版本更新机制失效。docker 部署的 nginx 模板已包含以下配置（见 `deploy/nginx/default.conf.template`）；非 docker / 自定义 nginx 部署需手动添加：

```nginx
location = /sw.js {
    add_header Cache-Control "no-cache, no-store, must-revalidate";
    expires off;
}
location = /manifest.webmanifest {
    add_header Cache-Control "no-cache";
    expires off;
}
```

### 3. 静态资源长缓存不变

带 hash 的 JS/CSS/图标仍走现有 30 天 immutable 长缓存规则，无需调整。

### 4. 验收

部署后用 Chrome DevTools → Lighthouse → PWA 类别审计，确认 **installable** 通过。
```

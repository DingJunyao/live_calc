# 菜谱发布默认私有 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 用户创建菜谱默认私有，点详情页「发布」按钮才公开（走审核）；发布对作者不可逆；已发布状态不再显示状态 chip，未发布才显示。

**Architecture:** 方案 A——「公共/已发布」一律以 `is_public` 为唯一准绳，`source` 回归来源标识本职。改后端创建点（自创默认私有 / 导入默认公共）+ 全部可见性判断（`source`→`is_public`）+ 数据迁移（导入菜谱补 `is_public=true`）+ 前端（`isPublished` 去 `source`、状态 chip、发布二次确认）。

**Tech Stack:** FastAPI + SQLAlchemy（后端）、Vue 3 + Vuetify（前端）、Alembic + 三引擎 SQL（迁移）。

**提交约定：** 遵循项目 CLAUDE.md，本计划**不含 git commit 步骤**。每个任务以「实现 + 验证通过」收口，最终是否提交由开发者决定。

const showPublishConfirm = ref(false)

const confirmPublish = async () => {
  if (!recipe.value) return
  publishing.value = true
  try {
    const result = await api.post(`/recipes/${recipeId.value}/publish`)
    showPublishConfirm.value = false
    if (result?.is_public) {
      notify('菜谱已发布', 'success')
      await loadData()
    } else if (result?.status === 'pending') {
      notify('发布提议已提交，待管理员审核', 'info')

      notify('发布请求已提交', 'info')
      await loadData()

  } finally {
    publishing.value = false
  }
}

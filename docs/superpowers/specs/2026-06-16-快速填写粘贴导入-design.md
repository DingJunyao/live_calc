# 设计文档：快速填写 - 粘贴文本导入价格

## 一、功能概述

在快速填写页面（QuickFillView）的导航栏上新增「粘贴导入」按钮，点击后弹出对话框。用户粘贴一段文本（从超市小票、备忘录等复制），系统解析出商品名、价格、数量、单位，逐条匹配商品并创建价格记录。不存在的商品由用户手动处理。

第一版为纯手动模式，不涉及 AI 智能匹配和后台 AI 配置（YAGNI，留给后续版本）。

## 二、交互流程

```
用户在快速填写页面（已选好商家）
    → 点击导航栏「粘贴导入」按钮
    → 弹出对话框：
        ┌───────────────────────────────────┐
        │  日期时间选择框（默认当前时间）        │
        │  大文本框（粘贴区域）                 │
        │  [解析] 按钮                        │
        ├───────────────────────────────────┤
        │  解析后展示预览表格：                │
        │  ┌────┬──────┬────┬────┬─────┐    │
        │  │状态│ 商品  │价格│数量│ 单位│    │
        │  ├────┼──────┼────┼────┼─────┤    │
        │  │ ✅ │ 芹菜  │1.88│ 1  │ 斤  │    │
        │  │ ⚠️ │ 芽菇  │4.00│ 1  │ 袋  │    │
        │  │ ❌ │ 荆芥  │5.99│ 1  │ 斤  │    │
        │  └────┴──────┴────┴────┴─────┘    │
        │                                    │
        │  ✅ 已匹配  ⚠️ 需确认  ❌ 未找到    │
```

## 三、解析规则

支持的文本格式（空格或 tab 分隔）：

| 格式 | 示例 | 解析结果 |
|------|------|---------|
| 名称 价格 | `芹菜 1.88` | 芹菜, 1.88元, 数量1, 单位「斤」 |
| 名称 价格/单位 | `芽菇 4/袋` | 芽菇, 4元, 数量1, 单位「袋」 |
| 名称 价格/kg | `嫩豆腐 5.18/kg` | 嫩豆腐, 5.18元, 数量1, 单位「kg」 |
| 名称 价格/数量+单位 | `土豆粉 2.5/200g` | 土豆粉, 2.5元, 数量200, 单位「克」 |

解析逻辑：
1. 按行拆分，忽略空行
2. 每行用空格/tab 分隔为「名称」和「价格部分」
3. 价格部分用 `/` 分隔为「价格」和「单位部分」
4. 单位部分提取末尾的非数字字符为单位名称，前面的数字为数量
5. 没有单位部分的，默认数量1、单位「斤」
6. 单位别名映射：系统单位的 abbreviation 为「克=g、千克=kg、斤=斤」，因此 `g`/`kg` 原样保留；中文别名归一化为缩写（`克→g`、`千克/公斤→kg`）；其余（如「袋」）原样交给后端，由 `UnitMatcher.match_or_create_unit` 自动匹配或创建
7. 无法解析的行标记为格式错误，展示在预览表格中

## 四、商品匹配（手动模式）

解析完成后，逐条用商品名去后端搜索现有商品：

- **精确匹配到商品** → 标记 ✅，可直接导入
- **未匹配到商品** → 标记 ❌，用户可操作：
  1. **关联已有商品**：点击商品名，打开搜索下拉，从已有商品中选择（如「水芹」→「水芹菜」）
  2. **创建新商品 + 同名原料**：用文本中的名字创建新原料，在新原料下创建同名商品（默认选项）
  3. **创建新商品 + 挂靠已有原料**：搜索选择已有原料（如选择「南瓜」），在其下创建商品「贝贝南瓜」
- 创建新商品时必须挂靠原料，不允许悬空
- 导入时只处理 ✅ 和已确认的行，未处理的跳过

### 商品与原料的关系

- 商品从属于原料；原料与菜谱相关
- 原料下面默认会有一个同名商品，但也不一定
- 原料记录营养成分数据，商品默认继承，也可覆盖
- 价格记录绑定到商品

## 五、进度与结果展示

**导入过程中：**
- 进度条 + 当前正在导入的商品名（如 `正在导入 3/10：嫩豆腐...`）
- 使用并发请求（并发数 5，复用快速填写的并发模式）

**导入完成后：**
- 成功 N 条，失败 M 条
- 失败的列出具体商品名和原因（网络错误、服务器报错等）
- 点关闭回到快速填写页面

## 六、技术要点

- **以前端为主，后端一处小改动**：
  - 后端 `ProductRecordCreate` 增加可选 `ingredient_id` 字段；`_get_or_create_product` 在提供 `ingredient_id` 时于指定原料下创建商品（用于「挂靠已有原料」场景）。该改动向后兼容（不传则维持原「同名原料+同名商品」行为）
  - 前端复用现有 API
- **API 调用**：
  - 商品搜索：`GET /products/autocomplete?q=&limit=` → `{id, name, ...}[]`
  - 原料搜索：`GET /ingredients?q=&limit=` → `{items: [{id, name, aliases}], total}`
  - 价格创建：`POST /products`，**不计入支出通过 `record_type='price'` 控制**（后端无 `include_in_spending` 字段；`purchase` 才计入支出）
  - 价格创建按场景传不同参数组合：关联已有商品传 `product_id`；创建同名原料+商品传 `product_name`；挂靠已有原料传 `product_name` + `ingredient_id`
- **商家**：沿用快速填写页面已选的商家
- **日期时间**：对话框内提供日期时间选择框，默认当前时间；保存时转 ISO 传 `recorded_at`
- **组件**：`PasteImportDialog.vue`，放在 `frontend/src/components/prices/` 下；解析逻辑抽为纯函数 `frontend/src/utils/pastePriceParser.ts`
- **状态**：导入完成后触发快速填写页面刷新，已导入的商品按现有逻辑隐藏
- **响应式**：对话框需适配移动端

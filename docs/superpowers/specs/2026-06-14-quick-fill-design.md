# 快速填写功能设计文档

日期：2026-06-14

## 概述

在价格记录页面增加「快速填写」入口，进入独立页面，允许用户先选择商家，然后以行为单位批量填写该商家各商品的价格，只保存填写了价格的商品。近 1 小时内已在当前页面填过的商品自动隐藏，避免重复录入。

## 设计范围

| 项目 | 范围 |
|------|------|
| 前端 | 新页面 QuickFillView.vue、新路由 `/prices/quick-fill`、app-bar 入口按钮 |
| 后端 | 无变更（复用现有 API） |
| 状态存储 | sessionStorage（按商家维度独立，1 小时过期规则） |

## 入口

### 按钮位置

在 `PricesView.vue` 的 `<v-app-bar>` → `<template #append>` 中，刷新按钮右侧添加一个图标按钮：

- 图标：`mdi-lightning-bolt`
- 点击导航至 `/prices/quick-fill`
- 纯图标按钮，不显示文字（与刷新按钮风格一致）

### 路由

在 `router/index.ts` 中 `/prices` 下添加子路由：

```
path: 'prices/quick-fill'
name: 'quick-fill'
component: QuickFillView.vue
meta: { title: '快速填写' }
```

## 页面结构

### 导航栏

```
[←] 快速填写            [💾]
```

- 左：返回箭头，点击回到 `/prices`
- 中：标题「快速填写」
- 右：保存按钮，纯图标 `mdi-check-all`，不显示文字

### 商家选择器

页面核心的入口控件，不选商家时填写区域为空：

- 使用 `v-autocomplete` 搜索选择商家
- 数据源复用 merchants store（`GET /merchants`）
- 选中商家后自动加载该商家下的历史商品列表
- 切换商家 → 清空当前行列表 → 按新商家加载数据 → 重置隐藏状态
- 选择器下方显示提示信息：「选商家后自动列出历史所有商品，只保存填了价格的」

### 商品行

选定商家后，页面分为两个区域：

#### 区域一：历史商品列表（按名称升序排列）

每行三个列：

```
商品名称（只读） | [价格输入框] | / 数量  单位
```

- **商品名称**：只读文本，不可修改
- **价格输入框**：数字输入，`step="0.01"`，placeholder `¥0.00`，始终显示为可编辑状态
- **数量**：默认以文本显示 `/ 1`，点击后变为数字输入框，填写后按 Enter 或失去焦点后恢复文本显示
- **单位**：默认以文本显示 `斤`，点击后变为单位下拉选择器，选择后恢复文本显示

数量默认值为 `1`，单位默认值为 `斤`。若用户未点击修改，提交时使用默认值。

#### 区域二：新增商品

- 新增行第一列为 `v-autocomplete`，搜索时调用 `GET /products/autocomplete`
- 搜索结果**排除**历史列表中已有的商品 ID（防止重复添加）
- 支持搜索无结果时创建新商品（复用现有 autocomplete 的「创建新商品」流程）
- 新增行的数量/单位交互同历史行（默认 `/ 1` `斤`，点击编辑）
- 新增行若未填写价格，行末显示 `✕` 删除按钮
- 底部有「＋ 添加新行」按钮，点击追加一行新增商品行

### 隐藏行提示

若当前商家下有 1 小时内已隐藏的商品，在历史商品列表底部显示：

```
── 近 1h 已填商品已隐藏（N 个） ──
```

## 保存逻辑

### 触发

点击导航栏右侧保存图标按钮 `mdi-check-all`。

### 收集规则

1. 遍历所有可见行（历史行 + 新增行）
2. 收集 `price` 不为空且 > 0 的行
3. 若新增行填了价格但商品为选择 → 提示「请选择第 N 行新增行的商品」
4. 若没有填写任何价格 → 提示「请至少填写一个商品的价格」

### 提交

逐条调用 `POST /products`：

```json
{
  "product_id": <商品ID>,
  "merchant_id": <选中商家ID>,
  "price": <填写价格>,
  "original_quantity": <数量，默认1>,
  "original_unit": <单位，默认"斤">,
  "record_type": "purchase"
}
```

待所有请求完成后：

| 结果 | 行为 |
|------|------|
| 全部成功 | snackbar「已保存 N 条」，写入 sessionStorage，隐藏已填行 |
| 部分失败 | snackbar「已保存 N 条，M 条失败」 |
| 全部失败 | snackbar「保存失败，请重试」 |

## 隐藏逻辑

### 存储

使用 sessionStorage，键值结构：

```
键: quick-fill-hidden-{merchantId}
值: [{ productId: number, filledAt: ISO string }]
```

### 规则

- **保存后即时隐藏**：保存成功的商品行立即写入 sessionStorage，并从页面列表中隐藏
- **按商家独立**：切换商家后重新加载该商家的隐藏列表，不互相影响
- **1 小时规则**：页面加载时读取 sessionStorage，过滤 `filledAt` 在当前时间 1 小时内的记录予以隐藏；超过 1 小时的忽略
- **跨页面访问生效**：退出快速填写页再进入，同样读取 sessionStorage 应用隐藏规则
- **手动刷新解除**：超过 1 小时后，用户刷新页面即可重新显示所有商品
- **新增行不参与隐藏**：隐藏规则仅对历史商品行生效，新增行不受影响

| 文件 | 变更 |
|------|------|
| `frontend/src/views/prices/PricesView.vue` | app-bar 追加区添加快速填写按钮 |
| `frontend/src/router/index.ts` | 添加 `/prices/quick-fill` 子路由 |
| `frontend/src/views/prices/QuickFillView.vue` | **新建**：快速填写页面主体 |
| `frontend/src/api/index.ts`（或相关 API 文件） | 无变更，复用现有方法 |

## 涉及的后端文件

无变更。复用以下现有 API：

- `GET /merchants` — 商家列表
- `GET /merchants/{id}/product-prices` — 商家各商品最新价格（取商品 ID 和名称列表）
- `GET /products/autocomplete` — 新增行商品搜索
- `POST /products` — 创建价格记录

## 边界情况

| 场景 | 处理 |
|------|------|
| 未选商家 | 填写区域显示空状态提示「请先选择商家」 |
| 商家无历史商品 | 历史区域显示「该商家暂无历史商品」 |
| 所有历史商品均已隐藏 | 历史区域显示「本期所有商品已填写完成」 |
| 新增行商品已在历史列表中存在 | autocomplete 返回结果排除该商品 |
| sessionStorage 写满 | 降级为仅本次 session 内生效，不报错 |

## 响应式设计

移动端优先：

- 每行占满宽度，价格输入框适当缩小适配小屏
- 数量/单位的文本点击区域适当放大，便于触控
- 历史商品区域可滚动
- 页面顶部导航栏固定

# 后台管理 UI 调整：合并翻译配置页 + 移除 USDA 数据页

- 日期：2026-06-21
- 状态：待实现
- 范围：前端（`frontend/src/views/admin/`、`frontend/src/router/index.ts`、`frontend/src/components/admin/`）

## 1. 背景与目标

后台管理当前存在三个可整理的页面：

- 「AI 配置」`AiConfigView.vue` 与「机翻配置」`MtConfigView.vue` 是两个独立页面；
- 「USDA 数据」`UsdaDataView.vue` 的功能已完全被「数据维护中心」`DataMaintenanceView.vue` 覆盖，属于冗余页。

本次调整三件事：

1. 将 AI 配置与机翻配置**合并为一个页面**，AI / 机翻分组展示、统一保存；
2. **移除**冗余的 USDA 数据页面；
3. **调整后台导航顺序**：合并后的配置页在前、数据维护中心紧随其后。

## 2. 现状分析

- `AiConfigView.vue` 与 `MtConfigView.vue` 共用同一个配置对象——`getTranslationConfig()` 返回 `{ ai: {...}, machine: {...} }`，二者各管一半；连 `testProvider` / `save` / `setField` 的逻辑都几乎一字不差地重复了一遍。合并本质是「物归原位」。
- `UsdaDataView.vue` 的全部功能（统计概览、下载 Foundation+SR Legacy、上传 zip、未映射营养素清单、任务状态轮询）已由 `DataMaintenanceView.vue` 100% 覆盖（参见其第 156–332、802–946 行的 USDA 数据管理区块、未映射营养素区块与任务列表）。
- 三个页面的引用面很小：仅 `router/index.ts` 的路由定义与 `AdminDashboard.vue` 的导航列表引用，别处无组件 `import`。

## 3. 设计决策

| 议题 | 决策 | 理由 |
|---|---|---|
| 合并页布局 | 可折叠面板 `v-expansion-panels multiple`，AI / 机翻两个面板**默认全展开** | 一屏看全，又可随手收起 |
| 合并页路由 | 复用 `/admin/ai-config`；`/admin/mt-config` 重定向到 `ai-config` | 旧书签不失效，改动最小 |
| 卡片组织 | 抽 `ProviderCard.vue` 子组件，用字段配置数组 `v-for` 渲染 6 张卡 | 消除重复模板，字段差异用配置表达（DRY） |
| USDA 数据页 | **直接删除**（文件 + 路由 + 导航项，不做重定向） | 功能已被数据维护中心覆盖，引用面小 |
| 导航顺序 | 邀请码 → 单位 → 地图 → **AI 与机翻配置** → **数据维护中心** → Agent 任务台 | 按用户要求，配置页在前、维护中心紧随 |

## 4. 详细设计

### 4.1 合并「AI 与机翻配置」页

#### 4.1.1 路由（`router/index.ts`）

- 保留 `admin/ai-config`，`component` 仍指向改造后的 `AiConfigView.vue`，`meta.title` 改为「AI 与机翻配置」。
- `admin/mt-config` 改为纯重定向：`redirect: '/admin/ai-config'`（与现有 `recipe-import → data-maintenance` 同一手法）。
- 删除 `admin/usda-data` 路由（见 4.2）。

#### 4.1.2 新增 `ProviderCard.vue` 子组件

位置：`frontend/src/components/admin/ProviderCard.vue`（新建 `components/admin/` 目录，与现有按功能分目录的惯例一致）。

**Props：**

| prop | 类型 | 说明 |
|---|---|---|
| `title` | `string` | 卡片标题，如「OpenAI 兼容」 |
| `fields` | `Array<{ key: string; label: string; type?: 'text' \| 'password' \| 'switch' }>` | 字段定义；缺省 `type` 按 `text` 处理 |
| `values` | `Record<string, any>` | 当前 provider 的字段值（含 `enabled`） |
| `testing` | `boolean` | 是否正在测试该 provider |
| `testResult` | `{ ok: boolean; detail: string } \| null` | 测试连接的就地反馈 |
| `hint` | `string` | 可选，卡片内的辅助说明 |

**Emits：**

| 事件 | 载荷 | 说明 |
|---|---|---|
| `update:field` | `{ field: string; value: any }` | 任一字段变更时上报 |
| `test` | — | 点击「测试连接」 |

**渲染：** `switch` 类型用 `v-switch`；`password` 用 `v-text-field` 配 `type="password"`；其余用 `v-text-field`。卡片底部一行：测试按钮 + 就地成功/失败文案（沿用现有 `testResult` 绿/红配色）。

**状态与逻辑**（两份重复逻辑合并为一套）：

- `config` / `testing` / `saving` / `saveMessage` / `testResult` 沿用现写法。
- `setField(group: 'ai' | 'machine', providerKey, field, value)` —— 比原来多一个 `group` 参数，写入 `config.value[group].providers[providerKey][field]`。
- `testProvider(providerKey)` —— 不变（provider key 在 AI/机翻间全局唯一）。
- `save()` —— 不变，一次 `putTranslationConfig(config.value)` 同时保存 AI 与机翻。
- `openPanels = ref([0, 1])` —— 控制两个面板默认全展开。

**模板骨架：**

```vue

          v-for="p in AI_PROVIDERS" :key="p.key"
```

### 4.2 删除 USDA 数据页

- 删除文件 `frontend/src/views/admin/UsdaDataView.vue`。
- `router/index.ts` 删除 `admin/usda-data` 路由条目（**直接删，不做重定向**）。
- `AdminDashboard.vue` 删除「USDA 数据」导航项。
- 功能无丢失：统计、下载、上传、未映射营养素、任务状态均已由数据维护中心覆盖。

### 4.3 调整后台导航顺序

`AdminDashboard.vue` 的 `v-list` 顺序调整为：

1. 邀请码管理
2. 单位管理
3. 地图设置
4. **AI 与机翻配置**（合并；图标用 `mdi-robot`，`to="/admin/ai-config"`，副标题「AI 服务与机器翻译密钥设置」）
5. **数据维护中心**（由原第 4 项下移）
6. Agent 任务台

`router/index.ts` 中 admin 子路由的书写顺序同步整理为与导航一致（非功能性，仅为整洁）。

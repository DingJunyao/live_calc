# 禁用地图总开关 · 设计文档

- 日期：2026-07-19
- 状态：已通过 brainstorming，待实现
- 涉及范围：后端（模型 / API / 迁移）+ 前端（商家列表 / 详情 / 表单 / 常用地点 / 管理员地图设置页）

## 1. 背景与目标

部分地图服务需要授权费用。需要一个**总开关**，让管理员可以一键禁用全部地图功能，从而：

- 不再产生地图调用、不暴露地图 UI
- 商家经纬度从必填变为可选，无坐标也能正常录入
- 个人常用地点功能随之隐藏（数据保留）

### 1.1 现状

- 地图配置存于 `map_configurations` 表（模型 `MapConfiguration`），含 `available_maps` / `default_map` / `map_api_keys` / `geocoding`。
- 管理员配置端点：`GET/PUT /admin/map-config`（仅管理员）。
- 公开配置端点：`GET /merchants/map-config`（任何登录用户），前端所有地图组件从此获取配置。
- 地图当前用于三处：商家列表（`MerchantsView`）、商家详情（`MerchantDetail`）、常用地点管理（`UserPlacesView`）。
- 商家经纬度 schema 本就是 `Optional`（`MerchantCreate.latitude/longitude`），后端 update 走 `model_dump(exclude_unset=True)`，前端不传则保留原值。
- 商家列表项坐标有效性判定：`isValidCoordinate(lat, lng)`（均为非 0 的 number）。
- 商家列表布局：桌面「左列表 350px + 右地图」、移动「列表在上 + 地图在下」。
- 价格管理布局（参考对象）：桌面卡片网格（`v-row + v-col` 响应式）、移动 `v-list lines="three"`，无地图。

## 2. 目标与非目标

### 2.1 目标

1. 管理员在「地图配置」页可设置总开关（启用 / 禁用地图）。
2. 禁用地图时：
   - 商家列表改为卡片网格（桌面）/ 列表（移动），样式参考价格管理，移除地图面板与定位按钮。
   - 商家新增 / 修改表单移除地图选点（MapPicker），经纬度变为不必填。
   - 已有经纬度的商家，修改时其坐标不受影响。
   - 无经纬度的商家，开启地图后修改时不会被赋予默认值或 0,0。
   - 开启地图时，无经纬度的商家不显示在地图上、其详情页也不显示地图。
   - 禁用地图时，所有商家详情页都不显示地图与位置区块。
3. 禁用地图时，个人不能维护常用地点：个人中心入口隐藏、路由拦截、后端写接口拒绝；数据保留。

### 2.2 非目标（YAGNI，本期不做）

- 不做配置变更的实时推送（管理员改开关后，其他页面 / 其他用户下次进入或刷新时生效）。
- 不拦截地理编码端点（后端辅助服务，保留能力）。
- 不改动 `MerchantMapView` / `MapPicker` 组件内部的配置获取逻辑（外层 `v-if` 已挡住渲染，不会触发）。
- 不清空或迁移已有 `user_places` 数据。
- 不改动导航 / 路由之外的其他地图相关入口（K 线图等页面未接入地图，不在范围内）。

## 3. 数据模型变更

### 3.1 字段方案选择

采用**方案 A：新增 `map_enabled` 布尔字段**（否决方案 B「复用 `available_maps` 空列表」）。

理由：

- 语义清晰：「要不要用地图」（`map_enabled`）与「用哪几张地图」（`available_maps`）正交。
- 临时关闭不丢失 `available_maps` 配置，重新启用原地恢复。
- `default_map` 联动逻辑不受影响（方案 B 在空列表时 `ensureDefaultMapValid` 不触发回退，`default_map` 残留无意义，是个小坑）。

### 3.2 模型改动

`backend/app/models/map_config.py` 的 `MapConfiguration`：

```python
import sqlalchemy as sa
# ...
map_enabled = Column(Boolean, nullable=False, default=True, server_default=sa.text("true"))
```

`to_dict()` 加入 `"map_enabled": self.map_enabled`。

注：`server_default` 用 `sa.text("true")` 而非 `sa.text("1")`，遵循项目既定规矩（见 `cc/BUGFIX_boolean_server_default_pg.md`），三库通吃。

### 3.3 迁移脚本

- alembic：`backend/alembic/versions/20260719_XXXX_add_map_enabled.py`，`add_column` + `server_default=sa.text("true")`。
- 四引擎 SQL 脚本（按 CLAUDE.md 惯例）：
  - `backend/scripts/sql/20260719_map_enabled_sqlite.sql`
  - `backend/scripts/sql/20260719_map_enabled_mysql.sql`
  - `backend/scripts/sql/20260719_map_enabled_postgresql.sql`
  - `backend/scripts/sql/20260719_map_enabled_postgresql_postgis.sql`（本变更与 PostGIS 无关，可省略此引擎或注明无关；按 CLAUDE.md「与 PostGIS 无关则不需要此项」处理）
- 开发库直接补（项目不走 alembic 惯例，`create_all` 不会给老表加列）。

## 4. 后端改动

### 4.1 管理员配置端点 `backend/app/api/admin.py`

- `MapConfig` Pydantic schema 加 `map_enabled: bool = True`。
- `DEFAULT_MAP_CONFIG` 字典加 `"map_enabled": True`。
- `update_map_config` 的 `config_data` 补 `"map_enabled": config.map_enabled`。

### 4.2 公开配置端点 `backend/app/api/merchants.py`

`GET /merchants/map-config`（`get_public_map_config`，当前在 `merchants.py:66`）：

- 返回体加入 `map_enabled`（从 `MapConfiguration` 读取）。
- 「无配置」默认分支的字典也补 `"map_enabled": True`。

前端所有地图相关页面通过此端点感知总开关状态。

### 4.3 常用地点写接口拦截 `backend/app/api/places.py`

在以下四个写端点入口读 `MapConfiguration.map_enabled`，为 `False` 时 `raise HTTPException(403, "地图功能已关闭，无法维护常用地点")`：

- `POST ""`（`create_user_place`）
- `PUT /{place_id}`（`update_user_place`）
- `DELETE /{place_id}`（`delete_user_place`）
- `PUT /{place_id}/default`（`set_default_user_place`）

`GET ""`（`list_user_places`）**不拦截**——数据保留可读，重新启用后前端可恢复使用。

实现建议：抽一个小 helper `_ensure_map_enabled(db)` 复用，避免四处重复查询逻辑。

## 5. 前端改动

### 5.1 地图配置统一获取 `frontend/src/composables/useMapConfig.ts`（新增）

模块级单例 composable（参考 `useTheme.ts` 套路），缓存 `/merchants/map-config` 结果：

- 暴露 `mapEnabled: Ref<boolean>`、`config: Ref<PublicMapConfig | null>`、`ensureLoaded(): Promise<void>`。
- 首次调用触发 fetch，后续共享缓存。
- 请求失败时 `mapEnabled` 默认回退 `true`（保守，不破坏现有功能）。
- 页面在 `onMounted` 调 `ensureLoaded()`。

`MerchantMapView` / `MapPicker` 内部的配置获取逻辑**不改**——它们只在 `mapEnabled=true` 时被渲染，照常工作。

### 5.2 商家列表 `frontend/src/views/data/MerchantsView.vue`

- 接入 `useMapConfig`，取 `mapEnabled`。
- 主内容区两套布局用 `v-if="mapEnabled"` / `v-else` 切换：
  - `mapEnabled=true`：保留现有「左列表 350px + 右地图」双栏。
  - `mapEnabled=false`：卡片网格（桌面 `v-row + v-col`，cols `12 / sm="6" / md="4" / lg="3" / xl="2"`），移动 `v-list lines="three"`，参考 `PricesView.vue:82-131`。
- 卡片 / 列表项内容：头像 + 名称、地址、坐标（有则显示、无则「未设置位置」灰字）、收藏 / 编辑 / 删除收三点菜单。
- 定位按钮（`locateMerchant`）在 `mapEnabled=false` 时隐藏。
- 搜索 / 筛选 / 分页 / FAB 全部保留。

### 5.3 商家表单（`MerchantsView` 的 `addDialog` + `MerchantDetail` 编辑对话框）

- `mapEnabled=false` 时，「商家位置」整块（含 `MapPicker`）`v-if` 隐藏。
- 提交逻辑：**禁用地图时，payload 完全不带 `latitude` / `longitude`**。即便 `pickerCoords` 因 `openEditDialog` 初始化带着原坐标，也不传入 payload。这样：
  - 编辑有坐标商家 → 后端 `exclude_unset` 保留原坐标，不受影响。
  - 新建 → `latitude` / `longitude` 为 `null`。
  - 编辑无坐标 → 仍为 `null`。
- 「开启地图后编辑无坐标商家不给默认值」：`mapEnabled=true` 时 `openEditDialog` 对无坐标商家保持 `pickerCoords=undefined`，`MapPicker` 不预填，用户不选则不传 → 保持 `null`。不在禁用 / 启用切换时偷塞默认值。

`MerchantDetail.vue:292` 的编辑表单 `MapPicker` 同样条件渲染 + 提交不带坐标。

### 5.4 商家详情 `frontend/src/views/merchants/MerchantDetail.vue`

- 地图区块（`MerchantMapView`，当前在 `:129`）显示条件收紧为 `mapEnabled && isValidCoordinate(当前商家)`：
  - 地图关闭 → 不显示。
  - 地图开启但当前商家无坐标 → 不显示。
- 位置信息区块（坐标文字 / 「未设置位置」提示，当前 `:75-90`）在 `mapEnabled=false` 时一并隐藏。

### 5.5 常用地点入口与页面

- `ProfileView.vue:97-106`「我的常用地点」入口 `v-if="mapEnabled"` 隐藏（`ProfileView` 接入 `useMapConfig`）。
- `router/index.ts:111` 的 `/profile/places` 路由加前置守卫：`mapEnabled=false` 时重定向回 `/profile`。
- `UserPlacesView.vue` 兜底：`mapEnabled=false` 时显示「地图功能已关闭」提示、`MapPicker` 不渲染、所有按钮 `disabled`，防止路由守卫与配置读取的竞态导致直接访问。

### 5.6 管理员地图设置页 `frontend/src/views/admin/MapSettingsView.vue`

- 顶部加 `v-switch`「启用地图功能」，`v-model="config.map_enabled"`。
- `MapConfig` TS interface 加 `map_enabled: boolean`，`config` reactive 初始值补 `map_enabled: true`。
- 关闭时：下方「可用地图 / API 密钥 / 地理编码」三块配置**灰显禁用**（`:disabled="!config.map_enabled"`，不隐藏），旁边提示文字「地图功能已关闭，前端所有地图隐藏，商家经纬度改为不必填。配置保留，重新启用后恢复」。
- 保存走既有 `PUT /admin/map-config`，`map_enabled` 随其他配置一起提交。

## 6. 边缘情况与行为约定

| 情况 | 处理 |
|---|---|
| `/merchants/map-config` 请求失败 | `useMapConfig` 默认 `mapEnabled=true`，不破坏现有功能 |
| 配置生效时机 | `mapEnabled` 在页面 `onMounted` 读一次、期间不变；管理员改开关后其他页面 / 用户下次进入或刷新时生效（低频操作，不做实时推送） |
| 禁用地图时 `MapPicker` / `MerchantMapView` | 外层 `v-if="mapEnabled"` 已挡，不渲染，内部 map-config 调用不触发 |
| 已有 `user_places` 数据 | 写接口挡、读接口不挡，数据完整保留，重新启用后即用 |
| 商家卡片网格里无坐标商家 | 照常显示，坐标位显示「未设置位置」，不参与过滤 |
| `available_maps` 与 `map_enabled` | 两者正交；关闭时 `available_maps` 配置保留，重新启用恢复 |
| 地理编码端点 | 不拦截，保留后端能力 |

## 7. 测试要点

### 7.1 后端

- `map_enabled=false` 时 `/places` 四个写接口（`POST` / `PUT` / `DELETE` / `PUT default`）均返回 403；`GET` 正常。
- `map_enabled=true` 时 `/places` 写接口正常（回归）。
- `GET /merchants/map-config` 返回体含 `map_enabled` 字段；默认分支也含。
- `PUT /admin/map-config` 可设置 `map_enabled`。
- `py_compile` 通过，既有商家 / 常用地点测试回归。

### 7.2 前端

- 商家列表：开关关闭下走卡片网格（桌面）/ 列表（移动），无地图面板、无定位按钮；开关开启下保留双栏（回归）。
- 商家表单：开关关闭下无 `MapPicker`，提交 payload 不含 `latitude` / `longitude`（编辑有坐标商家后端保留原值、新建为 null）。
- 商家详情：开关关闭或商家无坐标时不显示地图与位置区块。
- 常用地点：开关关闭下个人中心入口隐藏、`/profile/places` 路由重定向、直接访问兜底提示。
- 管理员设置页：总开关可切换、关闭时配置区灰显、保存生效。
- `npm run build` 通过。

## 9. 无表结构破坏性变更说明

仅新增一个带 `server_default=true` 的布尔列，老数据自动获得 `true`（启用），行为与现状一致，不影响存量商家与常用地点数据。无外键 / 约束 / 删除列变更。

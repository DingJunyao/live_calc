# 未使用图片清理增强设计

日期：2026-07-22
范围：后台「未使用图片清理」功能增强——支持头像路径、`users.avatar`/`Product.image_url` 引用、S3 存储后端列举。

## 背景

后台「未使用图片清理」（`GET /admin/images/unused`）存在三处不足：

1. **只扫 `recipes/` 目录**，不扫 `avatars/` 等子目录。换头像后旧头像文件无法出现在清理列表。
2. **只对照 `Recipe.images`** 字段判断「已使用」，忽略 `users.avatar`、`Product.image_url` 等图片字段。旧头像不会被识别为「未使用」。
3. **S3 后端时列举全空**：当前走本地 `iterdir()`（`admin.py:272`），S3 后端本地目录无文件。

DELETE 端点比列举好一些——[admin.py:304](backend/app/api/admin.py#L304) 用了 `storage.delete(key)`，已走存储后端。但硬编码 key 前缀 `recipes/`，无法处理其他子目录的文件。

## 设计

### 改动 1：StorageBackend protocol 加 `list` 方法

[base.py](backend/app/services/storage/base.py) `StorageBackend` protocol 新增：

```python
def list(self, prefix: str = "") -> list[str]:
    """列出给定 prefix 下的所有 key（递归）。返回空 list 表示无文件。"""
    ...
```

### 改动 2：LocalBackend 实现 list

[local.py](backend/app/services/storage/local.py) 新增 `list`：

- `self.images_dir / prefix` 下递归 walk 所有文件
- 返回相对 `self.images_dir` 的 key（如 `recipes/a.jpg`、`avatars/b.png`）
- 注意 `_path` 有路径穿越防御（不调用 `_path` 做 walk，直接 walk 后取相对路径）

### 改动 3：S3Backend 实现 list

[s3.py](backend/app/services/storage/s3.py) 新增 `list`：

- 复用 `client.list_objects_v2` 分页逻辑（migrate.py 已有相同模式）
- 只返回 Key（不含 ETag、Size 等，调用方如需统计信息可自己加——YAGNI）
- Prefix 兼容空和带 `/` 末尾两种输入

### 改动 4：GET /admin/images/unused 改造

[admin.py:247-283](backend/app/api/admin.py#L247) 重构：

1. **收集「已使用」key 集**（`set[str]`）：
   - `Recipe.images`（JSON list，每项是 key，如 `recipes/xxx.jpg`）
   - `users.avatar`（单 key，如 `avatars/xxx.png`，当非 None 时加入）
   - `Product.image_url`（单值，当非 None 且不以 `http` 开头时视为内部 storage key 加入）
2. **使用 `storage.list("")` 列出所有文件**（不再硬编码 `recipes/` 子目录）
3. **全集 − 已引用 = 未使用**，返回结构改为：

```json
{"images": [
  {"key": "recipes/麻婆豆腐_0.jpeg", "filename": "麻婆豆腐_0.jpeg", "size": 12345, "modified_at": "...", "url": "..."},
  {"key": "avatars/old_uuid.png", "filename": "old_uuid.png", "size": 6789, "modified_at": "...", "url": "..."}
]}
```

### 改动 5：DELETE 端点改造

[admin.py:286-308](backend/app/api/admin.py#L286)：

- 接受 `{"keys": ["recipes/xxx.jpg", "avatars/yyy.png"]}` 替代原来的 `{"paths": [...]}`
- 对每个 key 调 `storage.delete(key)`（不再硬拼 `recipes/` 前缀）
- 返回 `{"deleted": [...], "errors": [...]}` 格式不变

### 改动 6：前端 UnusedImagesView 改造

[UnusedImagesView.vue](frontend/src/views/admin/UnusedImagesView.vue)：

- 按子目录分组展示（「📁 菜谱图片」「📁 头像」），加 `v-divider` 分割
- 图片卡片显示完整 key（代替原来只显示 filename）
- 选中项存完整 key，删除时传 `keys: [...]`
- 兼容空结果（无未使用图片）显示
- 删除成功提示含各组删除数

### 不变（YAGNI）

- 不变更表结构
- 不扫其他非图片路径
- 不清理空目录
- 不修改 Product/User 存储路径的逻辑
- 不处理 Product.image_url 的外部 URL（不以 `http` 开头才作为内部 key）

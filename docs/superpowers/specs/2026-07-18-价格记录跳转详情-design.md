# 价格记录页点击跳转详情 — 设计文档

> 日期：2026-07-18
> 状态：待实现

## 背景

价格记录页（[PricesView.vue](frontend/src/views/prices/PricesView.vue)，路由 `/prices`）以移动端列表 / 桌面端卡片网格两种形态展示用户的价格记录。每条记录现成携带 `product_id`/`product_name` 与 `merchant_id`/`merchant_name`（[PriceRecord 接口 :326-339](frontend/src/views/prices/PricesView.vue#L326-L339)），但当前列表项**整行/整卡不可点击**，用户想看某条记录对应的商品或商家详情，只能手动去「数据管理」里翻找。

现状（均已代码层确认）：

- 移动端 `v-list-item`（[PricesView.vue:45](frontend/src/views/prices/PricesView.vue#L45)）：仅 append 区三个按钮（再记/编辑/删除），无整行跳转
- 桌面端 `v-card`（[PricesView.vue:87](frontend/src/views/prices/PricesView.vue#L87)）：仅 card-actions 三个按钮，无卡片跳转
- 顶部已用过 `$router.push`（[PricesView.vue:8](frontend/src/views/prices/PricesView.vue#L8)），项目跳详情模式成熟——商家管理列表即「整行 `@click` + `router.push` + 按钮区 `.stop` 防冒泡」范式（[MerchantsView.vue:57](frontend/src/views/data/MerchantsView.vue#L57) 整行 `@click`、[:551](frontend/src/views/data/MerchantsView.vue#L551) `router.push`、[:101](frontend/src/views/data/MerchantsView.vue#L101) 定位按钮 `.stop`）

## 目标

让价格记录页的两类实体可一键下钻到对应详情：

1. **点记录整体（空白处）→ 商品明细**（路由 `product-detail`，`/data/products/:id`）
2. **点记录内商家 → 商家详情**（路由 `merchant-detail`，`/data/merchants/:id`）

且满足：

- **三层点击互不打架**：整行（商品）/ 行内商家 / 按钮区（再记·编辑·删除），各用 `.stop` 隔离
- **无 id 容错**：未知商家（`merchant_id` 缺失）不可点、不报错、视觉降级
- **状态保持**：跳详情再返回，分页/搜索状态可恢复（现有 URL 同步机制兜底，无需额外开发）
- **零后端改动**、**不引新依赖**

## 设计

### 1. 新增两个跳转方法

挂在 `<script setup>` 既有 `router`（[PricesView.vue:322](frontend/src/views/prices/PricesView.vue#L322)）上，字符串路径风格对齐 [MerchantsView.vue:551](frontend/src/views/data/MerchantsView.vue#L551)：

```ts
// 点记录整体 → 商品明细
const goToProduct = (record: PriceRecord) => {
  if (!record.product_id) return
  router.push(`/data/products/${record.product_id}`)
}

// 点记录内商家 → 商家详情
const goToMerchant = (record: PriceRecord) => {
  if (!record.merchant_id) return
  router.push(`/data/merchants/${record.merchant_id}`)
}
```

两方法均带 guard，防御后端异常返回（`product_id` 接口标注必填、`merchant_id` 可选，仍统一兜底）。

### 2. 移动端 v-list-item（[PricesView.vue:45](frontend/src/views/prices/PricesView.vue#L45)）

整行加 `@click`，商家区与按钮区各自 `.stop`：

```vue
<v-list-item v-for="record in records" :key="record.id"
             @click="goToProduct(record)">
  ...
  <!-- 商家行（原 :57）改为可点 span -->
  <v-list-item-subtitle>
    <span v-if="record.merchant_id"
          class="text-primary cursor-pointer"
          @click.stop="goToMerchant(record)">
      {{ record.merchant_name }}
    </span>
    <span v-else class="text-medium-emphasis">未知商家</span>
  </v-list-item-subtitle>
  ...
  <template #append>
    <!-- 三按钮均补 .stop（原 :64-66） -->
    <v-btn icon="mdi-tag-multiple" ... @click.stop="openRecordAgainDialog(record)" />
    <v-btn icon="mdi-pencil"       ... @click.stop="openEditDialog(record)" />
    <v-btn icon="mdi-delete"       ... @click.stop="deleteRecord(record.id)" />
  </template>
</v-list-item>
```

v-list-item 整行点击天然带 ripple 水波纹反馈，无需额外点击样式。

### 3. 桌面端 v-card 网格（[PricesView.vue:87](frontend/src/views/prices/PricesView.vue#L87)）

整卡加 `@click` + `cursor-pointer` 视觉提示，card-actions 按钮 `.stop`：

```vue
<v-card elevation="0" class="list-grid-card cursor-pointer"
       @click="goToProduct(record)">
  <v-card-text>
    ...
    <!-- 商家 chip（原 :99-102）条件可点：方法内 guard，视觉按 id 区分可点性 -->
    <v-chip size="x-small"
            :color="record.merchant_id ? 'primary' : 'default'"
            :variant="record.merchant_id ? 'tonal' : 'outlined'"
            @click.stop="goToMerchant(record)">
      <v-icon start size="x-small">mdi-store</v-icon>
      {{ record.merchant_name || '未知商家' }}
    </v-chip>
    ...
  </v-card-text>
  <v-divider />
  <v-card-actions>
    <v-spacer />
    <!-- 三按钮均补 .stop（原 :111-113） -->
    <v-btn icon="mdi-tag-multiple" ... @click.stop="openRecordAgainDialog(record)" />
    <v-btn icon="mdi-pencil"       ... @click.stop="openEditDialog(record)" />
    <v-btn icon="mdi-delete"       ... @click.stop="deleteRecord(record.id)" />
  </v-card-actions>
</v-card>
```

桌面 chip 始终绑 `@click.stop="goToMerchant(record)"`，无 id 场景由方法内 guard 拦截；视觉用 color/variant 区分可点性（有商家 `primary tonal`、未知商家 `outlined default`）。`cursor-pointer` 给整卡 hover 手型提示。

### 4. 三层点击隔离矩阵

| 点击位置 | 触发动作 | 冒泡处理 |
|---|---|---|
| 整行/整卡空白 | `goToProduct` → 商品详情 | —（最外层接收） |
| 行内商家（移动 span / 桌面 chip） | `goToMerchant` → 商家详情 | `.stop` 拦截，不冒泡到整行 |
| 按钮区（再记 / 编辑 / 删除） | 各自原有操作 | `.stop` 拦截，不冒泡到整行 |

### 5. 状态保持（无需额外开发）

分页/搜索本就同步到 URL（[syncToUrl :547](frontend/src/views/prices/PricesView.vue#L547)），初始化读 `route.query`（[PricesView.vue:361-363](frontend/src/views/prices/PricesView.vue#L361-L363)）。跳详情再按浏览器返回，页码/搜索词/筛选自动恢复。

## 不做（YAGNI）

- ❌ **不改 QuickFillView**：快速填写是「选定商家后批量填价」场景，整页绑定单一商家，不存在「点记录商家」语义；点行的交互是填价格、不是看详情
- ❌ 不动后端 API、不动路由（`product-detail` / `merchant-detail` 均已存在）
- ❌ 不加 hover 浮起/阴影动画：cursor 手型 + ripple 已足够提示可点性
- ❌ 不抽公共「可点列表项」组件：当前仅一页一处需求，DRY 临界点未到

# 原料黑名单功能设计文档

**日期**: 2026-06-26
**状态**: 已确认

## 概述

用户可将不想要的原料加入个人黑名单。含黑名单原料的菜谱默认在浏览页隐藏，也不会出现在每日饮食推荐中。

配置入口在个人中心。同时提供过敏原分组快速选择（管理员维护分组 + AI 辅助匹配原料）。

---

## 需求范围

- **菜谱浏览页**：默认隐藏含黑名单原料的菜谱，提供开关切换
- **每日饮食推荐**：排除含黑名单原料的菜谱
- **菜谱详情页**：不做额外处理（用户可能从搜索/收藏等入口进来）
- **原料/商品管理页**：不做额外限制，保持原样
- **快速选择**：管理员维护过敏原分组，用户可一键将整个分组的原料加入黑名单

## 数据模型

### `user_ingredient_blacklist` — 用户原料黑名单

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | Integer PK | |
| `user_id` | FK → users.id | 用户 |
| `ingredient_id` | FK → ingredients.id | 屏蔽的原料 |
| `reason` | String(100), nullable | 可选原因（"过敏"、"不爱吃"等） |
| `source` | String(50), default="manual" | 添加来源：`manual` / `allergen_group` |
| `allergen_group_id` | FK → allergen_groups.id, nullable | 来源分组 |
| + AuditMixin | | 含 `created_at`, `is_active` |

**约束：** `UniqueConstraint(user_id, ingredient_id)`

### `allergen_groups` — 过敏原分组（管理员维护）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | Integer PK | |
| `name` | String(50) | 分组名，如"甲壳类"、"花生" |
| `display_order` | Integer, default=0 | 排序 |
| `is_active` | Boolean, default=True | 启用/禁用 |
| + AuditMixin | | |

### `allergen_group_ingredients` — 分组-原料映射

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | Integer PK | |
| `group_id` | FK → allergen_groups.id | |
| `ingredient_id` | FK → ingredients.id | |
| `is_ai_matched` | Boolean, default=False | 是否由 AI 匹配 |
| + AuditMixin | | |

**约束：** `UniqueConstraint(group_id, ingredient_id)`

## 后端 API

### 用户黑名单 API（`/blacklist`）

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/v1/blacklist` | 获取当前用户黑名单列表（分页，支持搜索） |
| `POST` | `/api/v1/blacklist` | 添加单个原料 `{ingredient_id, reason?}` |
| `DELETE` | `/api/v1/blacklist/{ingredient_id}` | 移除单个原料 |
| `POST` | `/api/v1/blacklist/batch` | 批量添加 `{ingredient_ids, reason?, allergen_group_id?}` |
| `DELETE` | `/api/v1/blacklist/batch` | 批量移除 `{ingredient_ids}` |
| `GET` | `/api/v1/blacklist/ingredient-ids` | 返回黑名单中所有 ingredient_id（轻量，供前端判断菜谱） |

### 过敏原分组 API（管理员）（`/admin/allergen-groups`）

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/v1/admin/allergen-groups` | 分组列表（含原料列表） |
| `POST` | `/api/v1/admin/allergen-groups` | 创建分组 |
| `PUT` | `/api/v1/admin/allergen-groups/{id}` | 编辑分组 |
| `DELETE` | `/api/v1/admin/allergen-groups/{id}` | 删除分组 |
| `POST` | `/api/v1/admin/allergen-groups/{id}/ingredients` | 添加原料到分组 |
| `DELETE` | `/api/v1/admin/allergen-groups/{id}/ingredients/{ingredient_id}` | 移除分组中原料 |
| `POST` | `/api/v1/admin/allergen-groups/{id}/ai-match` | 触发 AI Agent 匹配原料 |

### 公开只读

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/v1/allergen-groups` | 获取启用分组及原料（用户端快速选择用） |

### 菜谱列表增强

`GET /api/v1/recipes` 新增参数：
- `exclude_blacklist_ingredients` (bool, default=false)：排除含当前用户黑名单原料的菜谱

### 推荐引擎增强

`meal_recommender.py` 的 `_get_candidate_pool()` 查询用户黑名单中原料涉及的菜谱 ID，加入 `exclude_recipe_ids` 排除。

## 前端设计

### 个人中心入口

ProfileView 设置列表新增：

```
🔴 原料黑名单
   已屏蔽 X 种原料
```

点击打开黑名单管理对话框。

### 黑名单管理对话框

**快速选择区（上方）：**
- 从 `GET /allergen-groups` 加载分组列表
- 每个分组显示为 chip 标签
- 已全部加入的分组灰显不可再点
- 点击 → `POST /blacklist/batch` 批量添加

**黑名单列表（下方）：**
- 搜索框 + 已屏蔽原料列表
- 每条显示：原料名、来源（手动 / 来自 XX 分组）、原因
- 删除按钮
- 手动搜索添加入口

### 菜谱浏览页

RecipesView 筛选区新增开关：

```
[👁 隐藏含黑名单原料的菜谱]（默认开启）
```

- 开启 → 请求 `exclude_blacklist_ingredients=true`
- 关闭 → 显示全部
- 状态记 localStorage

### 管理员端 — 过敏原分组管理

新页面 `/admin/allergen-groups`：
- 分组列表（名称、原料数、状态、排序）
- 创建/编辑分组对话框
- 分组详情：原料列表 + 添加/移除 + **[AI 匹配]** 按钮
- 后台导航新增"过敏原分组"入口

---

## AI Agent 匹配机制

管理员点击分组（如"甲壳类"）的 **[AI 匹配]** 按钮：

1. 后端创建 Agent 任务（复用 `agent_sessions` + langchain Agent 机制）
2. Agent 通过只读 MCP 搜索数据库中与分组名语义匹配的原料
3. 匹配结果写入 `allergen_group_ingredients`，标记 `is_ai_matched=True`
4. 前端轮询任务状态，完成后刷新

**人工校验：** AI 匹配后管理员可手动移除误匹配或添加遗漏。AI 仅辅助提速。

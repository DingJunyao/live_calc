## 一、背景与目标

2026-07-20 的存储配置后台化 + 个人信息编辑重组上线、真机（阿里云 OSS）验证后，发现 6 个收尾点：

1. **头像上传缺裁剪**：当前 `POST /auth/me/avatar` 直接存原图，尺寸/构图不一。需上传前裁剪缩放，统一 **512×512**。
2. **国家/地区下拉显示英文**：ProfileView 级联国家级（level 0）显示英文，中国各级（省/市/县）是中文。根因是国家级数据源（[lukes/ISO-3166 英文 name](../../../backend/app/services/region_seed.py)）本身英文，非字段取错。
3. **存储切换向导两组前后退按钮**：StorageConfigView 的 `v-stepper` 自带 PREVIOUS/NEXT 与自定义中文「上一步/下一步」重复，需删自带的。
4. **缺 PicGo 风格自定义**：S3 配置无「存储路径前缀 / 自定义域名 / 网址后缀」，无法做 CDN 域名、子目录组织、图床处理参数（如七牛 `?imageslim`）。
5. **OSS 虚拟主机风格无提示**：向导② url_style 字段无说明，用户不知 OSS/AWS 该选 virtual 还是 path。
6. **阿里云 OSS PutObject 报 NotImplemented**：`STREAMING-UNSIGNED-PAYLOAD-TRAILER is not supported`——boto3/botocore 新版默认对 PutObject 用 streaming-unsigned-payload-trailer 签名，OSS 不支持。

### 目标

1. **头像裁剪**：vue-advanced-cropper 裁剪对话框，1:1 裁剪缩放，输出 512×512 上传。
2. **国家级中文化**：region_seed 内置 ISO→中文映射，re-seed 后国家级 name 为中文。
3. **向导按钮去重**：v-stepper 关默认 actions，留中文按钮。
4. **PicGo 三字段**：StorageConfiguration + S3Backend 支持 base_path / custom_domain / url_suffix。
5. **url_style 提示**：向导②加 OSS/MinIO 风格说明。
6. **OSS 兼容修复**：S3Backend boto3 Config 关 streaming-unsigned-payload-trailer。

### 非目标（YAGNI）

- 头像裁剪不做旋转/多比例（固定 1:1 输出 512×512）。
- PicGo 三字段仅 S3 模式生效（local 模式继续用 storage_base_url，不接入）。
- 不改 2026-07-20 已实现的三层优先级 / 双向迁移 / 向导五步主流程。

## 三、设计

### 3.1 头像裁剪（vue-advanced-cropper）

**新增依赖**：`vue-advanced-cropper`（Vue 3 原生兼容、维护活跃、支持固定比例 + 输出尺寸）。

**新组件 `frontend/src/components/profile/AvatarCropperDialog.vue`**：
- props：`modelValue`（开关）/ `src`（原图 ObjectURL）
- 内部：`<Cropper :src="src" :stencil-props="{ aspectRatio: 1 }" />`（1:1 裁剪框，可缩放/拖动）
- 确认按钮 → 用 `@advanced-cropper` 的 canvas 预览，`canvas.toBlob(cb, 'image/jpeg', 0.9)` 输出 **512×512**（resize 到 512）→ emit `cropped(blob)`
- 取消按钮 → emit close

**ProfileView 接入**：
- `triggerAvatarUpload` 选文件后，先 `URL.createObjectURL(file)` → 打开 `AvatarCropperDialog`（而非直接 uploadAvatar）
- 对话框 `@cropped` → `uploadAvatar(croppedBlob)`（复用现有上传逻辑，blob 作为 File 传给 FormData）
- 裁剪后的 512×512 blob 走 `POST /auth/me/avatar`，后端零改动

### 3.2 国家级中文化

**[`region_seed.py`](../../../backend/app/services/region_seed.py) 加常量**：
```python
# ISO 3166-1 alpha-2 → 中文名（~250 条，内置静态；缺失时回落英文 name）
COUNTRY_NAMES_ZH: dict[str, str] = {
    "CN": "中国", "US": "美国", "JP": "日本", "KR": "韩国",
    "GB": "英国", "FR": "法国", "DE": "德国", "RU": "俄罗斯",
    # ... 完整 ~250 条（ISO 3166-1 官方成员）
}
```
（数据来源：ISO 3166-1 官方国名中文译名表，内置常量保证不依赖外部网络源。）

**`_process_countries` 改**：
```python
name_zh = COUNTRY_NAMES_ZH.get(alpha2, name)   # 缺中文回落英文
_id, action = _upsert_region(
    db, code=alpha2, name=name_zh, level=0,
    iso_country=alpha2, path=alpha2, name_en=name,  # 英文存 name_en
)
```

**re-seed 触发**：复用 [`POST /admin/regions/seed`](../../../backend/app/api/regions.py)（2026-07-20 Task 2 端点），调 `upsert_administrative_regions` → 国家级按 code 去重 upsert，更新现有英文 name 为中文（`_upsert_region` 已存在则更新 name）。数据维护中心「更新行政区划数据」按钮即可生效，无需新端点。

### 3.3 向导按钮去重

[`StorageConfigView.vue`](../../../frontend/src/views/admin/StorageConfigView.vue) 的 `v-stepper` 加 `:hide-actions="true"`（Vuetify 3 关默认 PREVIOUS/NEXT），只留 implementer 自定义的中文「上一步/下一步」按钮组（已在向导底部）。

### 3.4 PicGo 三字段（S3 专用）

**`StorageConfiguration` 加 3 字段**（nullable）：
```python
s3_base_path = Column(String(255), nullable=True)       # key 前缀，如 "livecalc/"
s3_custom_domain = Column(String(255), nullable=True)    # 自定义域名，如 "https://cdn.example.com"
s3_url_suffix = Column(String(255), nullable=True)      # 网址后缀，如 "?imageslim"
```
+ alembic 迁移 + SQLite/MySQL/PostgreSQL 三引擎 SQL + 开发库直补。

**`S3Backend.__init__` 加 3 参**（默认空串/None）：
```python
def __init__(self, ..., base_path: str = "", custom_domain: str = "", url_suffix: str = ""):
    ...
    self.base_path = (base_path or "").strip("/")
    self.custom_domain = (custom_domain or "").rstrip("/")
    self.url_suffix = url_suffix or ""
```

**`put`**：key 加前缀
```python
def put(self, key, data, content_type):
    full_key = f"{self.base_path}/{key}" if self.base_path else key
    self.client.put_object(Bucket=self.bucket, Key=full_key, ...)
    return key   # 返回逻辑 key（不含前缀），DB 存逻辑 key
```
（**注意**：DB 仍存逻辑 key（如 `avatars/xxx.png`，不含 base_path）；`exists`/`get`/`delete` 内部自动加前缀查；`url_for` 加前缀拼 URL。这样 base_path 改动时 DB 零改动，迁移只重传文件。）

**`url_for`**：custom_domain 优先 + 前缀 + 后缀
```python
def url_for(self, key):
    full_key = f"{self.base_path}/{key}" if self.base_path else key
    quoted = quote(full_key)
    if self.custom_domain:
        base = self.custom_domain
    elif self.url_style == "virtual":
        parsed = urlparse(self.endpoint)
        scheme = parsed.scheme or "https"
        netloc = parsed.netloc or parsed.path
        base = f"{scheme}://{self.bucket}.{netloc}"
    else:
        base = f"{self.endpoint}/{self.bucket}"
    return f"{base}/{quoted}{self.url_suffix}"
```

**`exists`/`get`/`delete`**：内部用 `_full_key(key)` 加前缀。

**全链跟进**：
- [`effective.py`](../../../backend/app/services/storage/effective.py) `_ENV_MAP` + `_DEFAULTS` 加 3 字段（bootstrap_s3_base_path 等）
- [`config.py`](../../../backend/app/config.py) `Settings` 加 `bootstrap_s3_base_path` / `bootstrap_s3_custom_domain` / `bootstrap_s3_url_style`（.env baseline）
- [`factory._build`](../../../backend/app/services/storage/factory.py) 传 S3Backend 3 新参
- [`storage_config.py`](../../../backend/app/api/storage_config.py) `StorageConfigResponse` / `StorageConfigUpdate` / `StorageConfigTest` 加 3 字段；`_probe_s3` 构造的临时 S3Backend 也传 3 参
- 前端 [StorageConfigView.vue](../../../frontend/src/views/admin/StorageConfigView.vue) 向导②加 3 输入框 + 配置展示卡显示

**local 模式忽略这三字段**（local 用 storage_base_url，S3Backend 不参与）。

### 3.5 url_style 提示

StorageConfigView 向导② url_style 字段加 `hint` 或下方说明文字：
> OSS / AWS S3 选「虚拟主机」（`<bucket>.<host>`）；MinIO / 自建选「路径风格」（`<host>/<bucket>`）。不确定时 OSS/AWS 选 virtual，自建选 path。

### 3.6 OSS PutObject 兼容修复

[`s3.py`](../../../backend/app/services/storage/s3.py) 的 `Config` 加参数关 streaming-unsigned-payload-trailer：
```python
config = Config(
    signature_version="s3v4",
    s3={
        "addressing_style": "virtual" if url_style == "virtual" else "path",
        "payload_signing_enabled": True,   # 强制签名，不用 streaming unsigned
    },
    request_checksum_calculation="when_required",    # 关默认 checksum 上传
    response_checksum_validation="when_required",    # 关默认 checksum 校验
)
```

**验证点（实现时必须用真实 OSS bucket 确认）**：
- `payload_signing_enabled=True` 强制用 `AWS4-HMAC-SHA256` 签名（计算 body SHA256），绕过 streaming-unsigned-payload-trailer。
- `request_checksum_calculation` / `response_checksum_validation` 是 boto3 2.x 新参数，控制 CRC32C 等默认 checksum（OSS 也可能不支持）。
- **具体哪几个参数组合让 OSS PutObject 真过，实现时用阿里云 OSS bucket 实测**——可能 `payload_signing_enabled=True` 单独就够，也可能要同时设 checksum 控制。spec 不锁死，留给实现验证 + 测试覆盖（mock + 真实 bucket 手测）。
- AWS S3 本身兼容所有这些签名模式，此修法对 AWS S3 无副作用。

## 六、关键决策记录

| 决策点 | 选择 | 理由 |
|---|---|---|
| 头像裁剪库 | vue-advanced-cropper | Vue 3 原生、活跃、支持固定比例 + 输出尺寸 |
| 头像输出 | 512×512 JPEG（质量 0.9） | 统一尺寸 + JPEG 压缩体积 |
| 国家级中文 | 内置 ISO→中文映射 + re-seed | DB 存中文前后端一致；不依赖外部源 |
| PicGo 字段 | base_path / custom_domain / url_suffix（S3 专用） | 对齐 PicGo；local 不接入（YAGNI） |
| key 范式 | DB 存逻辑 key（不含 base_path），S3Backend 内部加前缀 | base_path 改动 DB 零改动 |
| 向导按钮 | v-stepper hide-actions + 留中文按钮 | 去重 |
| OSS bug 修法 | boto3 Config payload_signing_enabled + checksum 控制 | 通用兼容修法，AWS S3 无副作用 |

## 七、风险与待定

- **OSS 兼容修法需真实 bucket 验证**（§3.6）：boto3 版本 + OSS 端行为有差异，spec 不锁死参数组合，实现时实测。用户需提供 OSS bucket 凭证协助验证。
- **COUNTRY_NAMES_ZH 数据完整性**（~250 条）：内置静态常量，需保证覆盖 ISO 3166-1 全部成员；缺失项回落英文 name（不阻断）。可从公开中文 ISO 国名表整理。
- **vue-advanced-cropper 体积 + 移动端手势**：组件 ~数十 KB；移动端双指缩放/拖动需测（库原生支持，但项目响应式要核）。
- **base_path 与既有 key 兼容**：现有 recipes/avatars 的 key 无前缀；切 base_path 后老 key URL 失效（需重传或迁移）。向导切 base_path 时提示「现有图需重新迁移」。

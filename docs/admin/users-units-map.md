# 用户 / 单位 / 地图

这篇管三块后台配置：用户与邀请码、单位管理、地图引擎配置。

## 用户管理

用户列表能看到所有用户（用户名、邮箱、角色、状态）。

**编辑用户**：

- 编辑表单**不含密码字段**（避免误改）
- 可改用户名、邮箱、角色（管理员/普通）、启用/禁用等

**重置密码**：

- 编辑表单旁有"**重置密码**"按钮，点开独立对话框，输两次新密码
- 重置后该用户在**所有设备上的登录立即失效**（基于 `token_version` 机制：改密码时版本 +1，旧 token 全部作废）
- 普通字段（用户名、邮箱）的修改**不会**踢人下线，只有改密码会

> 改密码作废 token 是安全设计——密码泄露后重置即可让盗号者掉线。

## 邀请码

`REGISTRATION_REQUIRE_INVITE_CODE=true` 时（见 [配置](config-init.md)），新用户注册必须填邀请码。邀请码在这管理：

- **创建 / 编辑**：
  - 过期时间（日期 + 时间，到分）
  - 最大使用次数（`max_uses`）
  - 启用/禁用
- **状态**：已用次数（`used_count`）/ 最大次数；过期自动失效
- **创建者**：记录是谁发的码

> 邀请码的过期时间按本地时区存储和显示。

## 单位管理

单位是共享数据（功能说明见 [单位与密度](../units-density.md)）。单位管理页：

- **标准单位**（千克、克、斤、升、毫升、个等基准单位）→ **受保护**，不可删、不可改基准换算（动基准会破坏所有历史数据）
- **普通单位**：可新增、修改换算关系、删除
- **能量单位**（kcal / kJ）属于**用户级偏好**（见 [个人中心](../profile.md#单位偏好)），不在单位管理里——每个用户自己选

改/删单位走提议-审核（管理员直写立即生效）。详见 [提议审核台](review.md)。

> 单位的换算关系（如 1 斤 = 500 克）维护在单位表里，全系统统一。

## 地图配置

生计兼容多种地图引擎（见 [商家与地图](../merchants-map.md#地图引擎)）。地图配置在这里统一管理：

- **各引擎 API 密钥**：高德 / 百度（GL）/ 腾讯 / 天地图 / OpenStreetMap 各填各的
- **默认引擎**：选一个作为系统默认
- **持久化**：配置存数据库，重启不丢

> 高德地图、百度地图、腾讯地图，填写相关密钥后，即可使用其官方 SDK；否则，系统使用其瓦片地图（模糊，且更新不及时）。天地图必须使用 API 密钥。
>
> 改完保存即生效，用户刷新页面就用新配置。地图引擎的坐标系差异由系统内部统一处理，配置时只管填密钥。

## 后台导航位置

这三块在后台侧边栏各自有入口（邀请码 → 单位 → 地图）。AI 与机翻配置在单独的 [数据维护中心](data-maintenance.md#ai-与机翻配置) 页。

# 数据维护中心

数据维护中心是管理员维护基础数据的统一入口（替代了旧的菜谱导入页），把导入、USDA、AI 后处理、任务列表都聚到一起。

## 总览

数据维护中心聚合三类任务的入口和列表：

- **导入任务**：菜谱/原料/营养的批量导入
- **Agent 任务**：AI Agent 的批量维护（见 [Agent 任务台](agent.md)）
- **USDA 任务**：USDA 数据的下载/上传

所有任务统一在"任务列表"里展示，点进去能跳到对应的工作台。

## 菜谱 / 原料导入

- **远程导入**：从 `DATA_REPO_URL`（HowToCook_json）拉取
- **本地路径导入**：从 `DATA_LOCAL_PATH` 读（复用 `.env` 配置，不再在页面手填目录路径，避免配置漂移）
- **触发时机**：首次启动自动导入（见 [配置与首次启动](config-init.md)）；之后在这里手动触发

> 系统只在数据库初始化时自动导入一次（判断"是否已有 `source=json_repo` 的菜谱"），后续重启不会重复。

## USDA 数据

USDA FoodData Central 是营养数据的来源（见 [营养分析](../nutrition.md)）：

- **下载**：拉取 USDA 的 Foundation + SR Legacy 原始数据，入库去重
  - 已做 WAF 兼容（完整浏览器 UA + 并发重试 + 可选代理），USDA 概率切断时多重试
- **上传**：管理已下载的 USDA 数据
- USDA 下载/上传作为独立任务类型（`usda_tasks`），纳入任务列表

> USDA 下载较慢（Foundation 几百条 + SR Legacy 七千多条），在任务列表里轮询进度。

## AI 后处理

导入数据后，可以用 AI/Agent 批量补全：

- **食材名翻译**：AI（Claude Code / OpenAI / Anthropic）或机翻（百度 / 阿里云 / DeepL）
- **营养素名翻译**：走 172 条预设映射表，不耗 AI
- **模糊量推测**：菜谱里"少许""适量"等推测成具体克数（Agent）
- **密度推测**：按食材名推测密度（Agent）

每项都支持**强制重新处理全部**（默认只处理未处理的；强制会重推已处理的）。

> 注意：强制重推在 Agent 路径下有几个历史 bug 已修（指令清晰、传 force、行数护栏放宽），但仍建议先用小批量试。

合并的配置页 `/admin/ai-config`（原 AI 配置 + 机翻配置两页合一）：

USDA 数据页已并入数据维护中心，不再单独存在。

## 任务列表

任务列表合并三个来源：

- `import_tasks`（导入任务）
- `agent_sessions`（Agent 会话）
- `usda_tasks`（USDA 下载/上传）

每条任务显示类型、状态、进度。点进去跳对应工作台：

- Agent 任务 → [Agent 任务台](agent.md)
- 导入/USDA 任务 → 该任务详情

> USDA 任务在后台跑（无法取消），任务列表里不显示取消按钮；Agent 任务可以插话/审批。
